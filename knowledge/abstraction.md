# 核心抽象

业务层面的关键抽象与领域概念。

## 核心概念定义

- **业务单元**：面向业务的独立交付单元��是租户可独立管理、部署的核心业务载体，其下包含1个代码仓库、若干部署计划。

- **部署计划**：完整的配置包，涵盖从代码构建到实例运行的全链路配置，包含 CI 配置、CD 配置和实例配置。

- **实例**：部署计划执行后生成的运行态实体，承载业务单元的运行时状态。

- **依赖**：实例运行所需的中间件与基础能力（如MySQL、Redis、消息队列等），为实例提供运行支撑。

- **资源**：支撑实例运行的基础设施，核心包括CPU、内存、存储、网络，是实例运行的硬件基础。

## 静态结构

```
业务单元
    │
    ├── 代码仓库（唯一）
    │
    └── 部署计划（多个）
            │
            ├── CI 配置（构建）
            ├── CD 配置（部署）
            │
            └── 实例配置
                    │
                    ├── 环境配置（dev/test/prod）
                    ├── 资源配额（CPU/内存）
                    └── 依赖绑定（中间件/能力）
```

## 动态流转

```
代码仓库 ──提供代码──> 部署计划 ──执行──> 实例（运行态）
                          │
                          ├── CI 配置 → 构建
                          ├── CD 配置 → 部署
                          └── 实例配置 → 运行
```

## 核心关系说明

1. **业务单元**是顶层业务抽象，包含1个代码仓库和多个部署计划；

2. **部署计划**是完整配置包，包含 CI 配置（构建）、CD 配置（部署）、实例配置（运行）；

3. **实例**是部署计划执行后的运行态产物，一个部署计划可多次执行生成多个实例；

4. 实例运行需绑定**依赖**（中间件能力）、占用**资源**（硬件支撑），一个依赖可被多个实例共享。

## 实体关系

| 实体 | 关系 | 目标实体 | 说明 |
|------|------|---------|------|
| 业务单元 | 1:1 | 代码仓库 | 每个业务单元对应一个代码仓库 |
| 业务单元 | 1:N | 部署计划 | 一个业务单元可有多个部署计划 |
| 部署计划 | 1:1 | CI 配置 | 每个部署计划包含一个 CI 配置 |
| 部署计划 | 1:1 | CD 配置 | 每个部署计划包含一个 CD 配置 |
| 部署计划 | 1:1 | 实例配置 | 每个部署计划包含一个实例配置 |
| 实例配置 | 1:1 | 环境配置 | 环境是实例配置的属性 |
| 实例配置 | 1:1 | 资源配额 | 资源是实例配置的属性 |
| 实例配置 | N:M | 依赖 | 实例配置可绑定多个依赖，依赖可被共享 |
| 部署计划 | 1:N | 实例 | 一个部署计划可多次执行生成多个实例 |
