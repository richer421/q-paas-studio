# 核心语义

## 项目

- **名称**: Q-PaaS Studio
- **定位**: PaaS 开发工作室，统一管理多个子服务的开发与部署
- **领域**: 云原生 PaaS 平台 / DevOps / 服务发布与元数据管理

## 系统边界

做什么：
- 子模块统一管理（q-deploy、q-ci、q-workflow、q-metahub）
- 知识库集中维护（架构、概念、定位）
- 开发规范与约定定义

不做什么：
- 具体业务逻辑实现（由各子模块负责）
- 运行时服务（各子模块独立运行）
- 直接的 CI/CD 执行（由 q-ci 负责）

## 子模块职责

| 子模块 | 职责 |
|--------|------|
| q-deploy | 服务发布平台，支持发布/更新/回滚操作 |
| q-ci | 持续集成服务，负责构建与测试 |
| q-workflow | 工作流引擎，编排自动化流程 |
| q-metahub | 元数据中心，管理服务配置与依赖 |
