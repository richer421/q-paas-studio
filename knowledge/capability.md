# 核心业务能力

描述系统解决的业务问题和核心能力边界。

## 设计原则

- 只描述"做什么"，不描述"怎么做"
- 接口细节由 Swagger 文档维护，此处不重复
- 新增模块时，说明业务场景而非 API 列表

## 能力清单

### q-deploy（服务发布平台）

核心发布能力，支持多种发布模式。

- **发布**：完整部署，包含新包产物 + 新配置
- **更新**：配置变更型发布，包产物不变
- **回滚**：回退到历史版本
- 渲染引擎（Helm/Kustomize/自定义模板）
- 工作引擎（Kubernetes/Docker/SSH）

### q-ci（持续集成服务）

构建与测试能力。

- 代码构建
- 测试执行
- 产物管理

### q-workflow（工作流引擎）

流程编排能力。

- 工作流定义
- 任务编排
- 状态追踪

### q-metahub（元数据中心）

元数据管理能力，为其他服务提供配置元数据支撑。

- **环境管理**：dev/test/prod 等部署环境的定义与查询
- **资源管理**：CPU/内存/存储/网络资源配额模板
- **依赖管理**：MySQL/Redis/MQ 等中间件依赖定义与绑定
- **实例配置**：组合环境+资源+依赖的元数据聚合
